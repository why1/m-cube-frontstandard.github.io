前端代码规范

概述

欢迎使用前端代码规范，使用过程中如碰到问题，请到Gitlab进行提问。

HTML规范

书写风格

HTML代码大小写 *（必要）

HTML标签名、类名、标签属性和大部分属性值统一用小写
HTML文本、CDATA、JavaScript、meta标签某些属性等内容可大小写混合

元素属性 *（必要）

  • 元素属性值使用双引号语法
  • 元素属性值可以写上的都写上

代码缩进 *（必要）

统一使用四个空格进行代码缩进，使得各编辑器表现一致（各编辑器有相关配置）

纯数字输入框 *（必要）

使用 type="tel" 而不是 type="number"

代码嵌套 *（必要）

元素嵌套规范，每个块状元素独立一行，内联元素可选

单行注释 *（必要）

一般用于简单的描述，如某些状态描述、属性描述等

注释内容前后各一个空格字符，注释位于要注释代码的上面，单独占一行

模块注释 *（必要）

一般用于描述模块的名称以及模块开始与结束的位置

注释内容前后各一个空格字符，<!-- S Comment Text --> 表示模块开始，<!-- E Comment Text --> 表示模块结束，模块与模块之间相隔一行

图片规范

PC平台单张的图片的大小不应大于 500KB。
移动平台单张的图片的大小不应大于 250KB。
上线的图片都应该经过压缩处理，压缩后的图片不应该出现肉眼可感知的失真区域
60质量的JPEG格式图片与质量大于60的相比，肉眼已看不出明显的区别，因此保存 JPEG 图的时候，质量一般控制在60，若保真度要求高的图片可适量提高到 80，图片大小控制在 0-500KB 以内

CSS/less规范

编码规范

团队约定 *（必要）

  • 样式文件必须写上 @charset 规则，并且一定要在样式文件的第一行首个字符位置开始写，编码名用 “UTF-8”
  • 字符 @charset “”; 都用小写字母，不能出现转义符，编码名允许大小混写
  • 考虑到在使用“UTF-8”编码情况下 BOM 对代码的污染和编码显示的问题，在可控范围下，坚决不使用 BOM。

BOM（Byte-Order Mark，字节序标记）是Unicode码点U+FEFF。它被定义来放在一个UTF-16文件的开头，如果字节序列是FEFF那么这个文件就是大端序，如果字节序列是FFFE那么这个文件就是小端序。BOM设计出来不是用来支持HTML和XML的。要识别文本编码，HTML有charset属性，XML有encoding属性，UTF-8 的网页代码不应使用 BOM，否则常常会出错。

代码格式化 *（必要）

统一使用展开格式书写样式

代码大小写 *（必要）

样式选择器，属性名，属性值关键字全部使用小写字母书写，属性字符串允许使用大小写。

代码缩进 *（必要）

统一使用四个空格进行代码缩进，使得各编辑器表现一致（各编辑器有相关配置）,默认一个tab键的距离

分号 *（必要）

每个属性声明末尾都要加分号；

代码易读性 *（必要）

左括号与类名之间一个空格，冒号与属性值之间一个空格

逗号分隔的取值，逗号之后一个空格

为单个css选择器或新申明开启新行

css属性值需要用到引号时，统一使用单引号

SCSS 文件内 *（必要）

全部遵循 CSS 注释规范
不使用 /*! */ 注释方式
注释内不放 SASS 变量
嵌套层级不超过四个层级

命名规范

ClassName的命名应该尽量精短、明确，必须以字母开头命名，且全部字母为小写，单词之间统一使用下划线 “_” 连接

模块命名 *（必要）

全站公共模块：以 mod_ 开头

JavaScript 规范

全部遵守ESlint,以及vue书写规范
