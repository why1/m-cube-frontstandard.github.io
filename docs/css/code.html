编码规范

团队约定 *（必要）

  • 样式文件必须写上 @charset 规则，并且一定要在样式文件的第一行首个字符位置开始写，编码名用 “UTF-8”
  • 字符 @charset “”; 都用小写字母，不能出现转义符，编码名允许大小混写
  • 考虑到在使用“UTF-8”编码情况下 BOM 对代码的污染和编码显示的问题，在可控范围下，坚决不使用 BOM。

BOM（Byte-Order Mark，字节序标记）是Unicode码点U+FEFF。它被定义来放在一个UTF-16文件的开头，如果字节序列是FEFF那么这个文件就是大端序，如果字节序列是FFFE那么这个文件就是小端序。BOM设计出来不是用来支持HTML和XML的。要识别文本编码，HTML有charset属性，XML有encoding属性，UTF-8 的网页代码不应使用 BOM，否则常常会出错。

代码风格

代码格式化 *（必要）

样式书写一般有两种：一种是紧凑格式 (Compact)

一种是展开格式（Expanded）

团队约定 *（必要）

统一使用展开格式书写样式

代码大小写 *（必要）

样式选择器，属性名，属性值关键字全部使用小写字母书写，属性字符串允许使用大小写。

代码缩进 *（必要）

统一使用四个空格进行代码缩进，使得各编辑器表现一致（各编辑器有相关配置）,默认一个tab键的距离

分号 *（必要）

每个属性声明末尾都要加分号；

代码易读性 *（必要）

左括号与类名之间一个空格，冒号与属性值之间一个空格
